## 怎么用这份清单

不用全装。按下面思路选：

1. 先确认自己缺什么能力——是需要 AI 连外部系统（装 MCP Server），还是需要 AI 换个专精角色写代码（装 Subagent），还是需要 AI 遵循一套固定工作方法（装 Skill）。
2. 每类先装 3-5 个高频会用的，不要一次性全装——工具/技能/代理越多，主对话判断"该用哪个"的成本越高，实际体验反而变差。
3. 装完先在一个小任务上验证真的生效，再用到正式工作里。

## MCP Server：官方参考实现

出自 [modelcontextprotocol/servers](https://github.com/modelcontextprotocol/servers)，Anthropic 官方维护，是接入 MCP 生态最稳的起点。

**当前维护中：**

| Server | 一句话 |
|---|---|
| Everything | 参考/测试用服务器，演示 prompts、resources、tools 三种能力 |
| Fetch | 抓取网页内容并转成适合喂给 LLM 的干净格式 |
| Filesystem | 安全的本地文件读写，可配置访问范围 |
| Git | 读取、搜索、操作 Git 仓库 |
| Memory | 基于知识图谱的跨对话持久记忆 |
| Sequential Thinking | 把复杂问题拆成可回溯、可修正的多步推理链 |
| Time | 时间与时区换算 |

**已归档但仍被社区广泛使用（部分已转交第三方维护）：**

| Server | 一句话 |
|---|---|
| GitHub | 仓库管理、文件操作、GitHub API 集成 |
| GitLab | GitLab API，支持项目管理 |
| Google Drive | Google Drive 文件访问与搜索 |
| Google Maps | 地图定位、路线规划、地点详情 |
| PostgreSQL | 只读数据库访问，支持 schema 查看 |
| Puppeteer | 浏览器自动化与网页抓取 |
| Redis | 读写 Redis 键值存储 |
| Sentry | 拉取并分析 Sentry.io 上的报错 |
| Slack | 频道管理与消息收发（现由 Zencoder 维护新版） |
| SQLite | 数据库交互与简单 BI 分析 |
| Brave Search | 网页与本地搜索（已有官方替代版本） |

## Claude Code Subagent：100+ 子代理，按方向分类

出自 [VoltAgent/awesome-claude-code-subagents](https://github.com/VoltAgent/awesome-claude-code-subagents)，共 10 大类、100+ 现成子代理。装法：`npx skills add` 对应插件包，或去仓库按需复制单个子代理的 Markdown 文件到 `.claude/agents/`。

| 分类 | 覆盖内容 | 举例 |
|---|---|---|
| 01 核心开发 | 日常编码通用角色 | api-designer、backend-developer、frontend-developer、fullstack-developer、mobile-developer |
| 02 语言专家 | 按语言/框架深度定制 | typescript-pro、golang-pro、rust-engineer、python-pro、nextjs-developer、django-developer |
| 03 基础设施 | DevOps / 云 / 部署 | cloud-architect、kubernetes-specialist、terraform-engineer、sre-engineer、security-engineer |
| 04 质量与安全 | 测试、审查、安全 | code-reviewer、security-auditor、penetration-tester、qa-expert、debugger |
| 05 数据与 AI | 数据工程、机器学习 | data-engineer、machine-learning-engineer、llm-architect、prompt-engineer |
| 06 开发者体验 | 工具链与效率 | cli-developer、documentation-engineer、mcp-developer、refactoring-specialist |
| 07 垂直领域 | 特定行业技术 | blockchain-developer、fintech-engineer、game-developer、iot-engineer |
| 08 业务与产品 | 产品/项目管理 | product-manager、business-analyst、technical-writer、ux-researcher |
| 09 多代理编排 | Agent 协作与调度 | multi-agent-coordinator、workflow-orchestrator、context-manager、agent-organizer |
| 10 研究与分析 | 调研、市场、竞品分析 | research-analyst、competitive-analyst、trend-analyst、market-researcher |

**挑选建议**：按你项目实际需要装一小簇（比如做后端就装 `backend-developer + code-reviewer + security-auditor`），不要整个合集全装——子代理太多会拖慢主对话判断该派给谁的速度。

## 新手 3 步选型建议

**第一步：装 3 个基础 Skill**

```bash
npx skills add anthropics/skills@frontend-design   # 设计质量
npx skills add obra/superpowers                    # 代码质量/工作流
npx skills add intellectronica/agent-skills@context7  # 文档查询
```

**第二步：按你的方向加 1-2 个**

| 方向 | 加这个 |
|---|---|
| 做 UI / 前端 | UI-UX-Pro-Max + Emil Kowalski's Skill |
| 做视频 | Remotion |
| 做营销页 | SEO Audit + Copywriting |
| 做组件库 | shadcn/ui + Tailwind CSS |
| 需要连外部系统 | 官方 MCP：filesystem / github / memory |

**第三步：用 Find Skills 让 Agent 自己找**

```bash
npx skills add vercel-labs/skills@find-skills
```

装完之后遇到没覆盖的场景，直接让 Claude 自己去搜索、安装合适的 Skill，不用每次手动翻合集。
